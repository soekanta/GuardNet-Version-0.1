ABSTRAK

Phishing merupakan salah satu ancaman keamanan siber terbesar yang menargetkan pengguna internet melalui tautan palsu yang menyerupai situs web sah. Penelitian ini bertujuan untuk mengembangkan sistem deteksi phishing berbasis Machine Learning yang diimplementasikan secara langsung (browser-side) dalam bentuk ekstensi Google Chrome bernama "GuardNet". Sistem ini menggunakan algoritma Logistic Regression yang dilatih dengan dataset PhiUSIIL untuk mengklasifikasikan URL berdasarkan 50 fitur ekstraksi yang mencakup karakteristik URL dan konten halaman. Model yang dikembangkan diintegrasikan menggunakan TensorFlow.js, memungkinkan analisis risiko secara real-time tanpa mengirimkan data pengguna ke server eksternal, sehingga menjaga privasi. Hasil pengujian menunjukkan bahwa ekstensi mampu mendeteksi situs phishing dengan akurasi yang signifikan dan memberikan peringatan visual berjenjang (Aman, Waspada, Bahaya) kepada pengguna.

Kata Kunci: Phishing, Logistic Regression, Chrome Extension, TensorFlow.js, Keamanan Siber.

1. Pendahuluan

Peningkatan aktivitas daring berbanding lurus dengan meningkatnya kejahatan siber, di mana serangan phishing menjadi metode yang paling umum digunakan untuk mencuri data sensitif pengguna. Metode konvensional berbasis daftar hitam (blacklist) seringkali lambat dalam mendeteksi ancaman baru (zero-day attacks). Oleh karena itu, diperlukan pendekatan heuristik berbasis kecerdasan buatan.

Penelitian ini mengusulkan implementasi model Logistic Regression yang ringan namun efektif, diletakkan langsung pada sisi klien (client-side) melalui ekstensi peramban. Pendekatan ini mengatasi masalah latensi jaringan dan privasi data yang sering ditemui pada solusi berbasis server.

2. Tinjauan Pustaka

2.1 Logistic Regression

Logistic Regression adalah algoritma klasifikasi statistik yang digunakan untuk memprediksi probabilitas kejadian suatu peristiwa logika binari. Dalam konteks ini, algoritma digunakan untuk menentukan probabilitas apakah sebuah URL termasuk kategori "Phishing" (0) atau "Legitimate" (1).

2.2 TensorFlow.js

TensorFlow.js adalah pustaka JavaScript yang memungkinkan pelatihan dan penggunaan model Machine Learning langsung di peramban web. Penggunaan teknologi ini memungkinkan GuardNet berjalan secara mandiri di dalam ekstensi Chrome.

3. Metodologi Penelitian

3.1 Pengumpulan Data

Data latih diambil dari dataset PhiUSIIL Phishing URL Dataset, yang mencakup ribuan sampel URL phishing dan sah. Dataset ini memiliki fitur-fitur yang kaya, meliputi properti leksikal URL dan karakteristik konten HTML.

3.2 Ekstraksi Fitur

Sistem mengekstrak 50 fitur numerik dari setiap URL yang kunjungi pengguna. Fitur-fitur tersebut dibagi menjadi dua kategori utama:

  • Fitur Berbasis URL (22 Fitur): Panjang URL, penggunaan IP address, panjang domain, entropi karakter, penggunaan HTTPS, dll.
  • Fitur Berbasis Konten (28 Fitur): Ketersediaan Favicon, keberadaan Title, rasio tautan eksternal, keberadaan form sensitif, dll.

3.3 Arsitektur Sistem

Sistem dibangun sebagai Ekstensi Chrome dengan arsitektur Manifest V3. Komponen utamanya terdiri dari:

  1. Background Service Worker: Memantau perpindahan tab dan URL.
  2. Sandbox Environment: Menjalankan model TensorFlow.js dan logika ekstraksi fitur di lingkungan terisolasi untuk keamanan.
  3. Popup UI: Antarmuka pengguna yang menampilkan skor risiko (0-100%) dan indikator warna (Hijau, Kuning, Merah).

4. Implementasi dan Hasil

Model yang telah dilatih diekspor ke format JSON yang kompatibel dengan web. Pada tahap inferensi, ekstensi melakukan normalisasi fitur menggunakan StandardScaler yang memiliki parameter sama dengan saat pelatihan.

4.1 Mekanisme Deteksi

Ketika pengguna mengunjungi sebuah situs, content_script mengambil struktur DOM halaman. Data ini dikirim ke sandbox untuk diekstrak menjadi vektor fitur. Model kemudian memprediksi skor probabilitas.

Hasil prediksi diinversikan (1 - skor_legitimasi) untuk mendapatkan "Skor Risiko Phishing".

4.2 Penanganan Kendala Teknis (CORS)

Salah satu tantangan utama dalam implementasi browser-side adalah kebijakan keamanan Cross-Origin Resource Sharing (CORS) yang mencegah pembacaan konten dari beberapa situs web. Untuk mengatasi ini, sistem menerapkan logika hibrida:

  • Jika konten dapat dibaca, fitur penuh digunakan untuk prediksi akurat.
  • Jika konten terblokir (CORS) namun URL menggunakan HTTPS dan domain terpercaya, sistem memberikan penalti risiko yang lebih rendah atau peringatan "Waspada" alih-alih vonis "Phishing", untuk mengurangi False Positive pada situs sah yang ketat.

5. Kesimpulan

Implementasi deteksi phishing menggunakan Logistic Regression pada ekstensi Chrome terbukti menjadi solusi yang layak dan efisien. "GuardNet" mampu memberikan perlindungan real-time dengan akurasi yang memadai untuk penggunaan sehari-hari. Pendekatan sisi klien berhasil menjaga privasi pengguna dan mempercepat proses deteksi.
